## Why This Exists

Robot manipulation is moving too fast for static reading lists.

Important ideas are scattered across thousands of papers. Strong work gets buried under volume. Trend shifts are easy to miss. This project turns that chaos into something you can actually use: a field map, a trend detector, and a fast filter for finding the papers that deserve your attention.

## What This Repository Covers

This repository tracks arXiv papers related to **robot manipulation** from **2023-01-01** up to the latest update time of the project.

- Paper discovery comes from **arXiv**
- Citation counts come from **Google Scholar** via cached lookup
- Keywords are generated by an **automated extraction workflow**

## Three Ranking Modes

The leaderboard supports three ranking views:

- **Overall**: total citation impact
- **Hot**: impact adjusted by recency
- **Newest**: publication date order

In practice:

- **Overall** highlights papers that have already shaped the field
- **Hot** surfaces papers that are rising fast right now
- **Newest** helps you watch the frontier move in real time

## Keyword Filtering

The site currently includes **76 keywords**.

You can combine them freely in the search box:

- `a + b`
- `a + b - c`

Examples:

- `VLA + world model`
- `reinforcement learning + robotic manipulation - survey`
- `tactile + dexterous manipulation`

This makes it easy to build your own slice of the field in seconds.

## Important Note on Keywords

Keywords in this project are intentionally conservative.

They aim to capture the **core contribution** of each paper, not every component mentioned in the abstract. A paper may use a benchmark, modeling trick, control element, or training ingredient related to a tag, but if that element is not part of the paper's main contribution, that tag may not be extracted.

This is a deliberate tradeoff: cleaner retrieval, sharper subfield ranking, and more meaningful search results.

## Score Formula

The ranking scores are computed from citation count and paper age.

Let:

- `cited_by_count` be the cached Google Scholar citation count
- `age_days` be the number of days since publication
- `lambda = 0.00075`

Then:

- `citation_factor = 1 + ln(1 + cited_by_count)`
- `freshness = exp(-lambda * age_days)`
- `Overall Score = citation_factor`
- `Hot Score = citation_factor * freshness`

Interpretation:

- **Overall Score** measures citation impact with logarithmic compression
- **Hot Score** rewards citation impact while discounting older papers
